## 04-01: Patient Dashboard - Uploaded Data List

### Objective
Create patient dashboard page showing user's uploaded neural data with CID, timestamp, and access management.

### Tasks

1. **Create usePatientData hook** (`src/lib/hooks/usePatientData.ts`)
   - Use `getDataCount(address)` to get user's upload count
   - Loop through data IDs and fetch each with `getData(dataId)`
   - Return array of `{ dataId, cid, timestamp, metadata }`
   - Handle loading and error states

2. **Create UploadedDataList component** (`src/components/UploadedDataList.tsx`)
   - Display table of uploaded data
   - Columns: ID, CID (truncated), Timestamp, Access Count
   - Click CID to view on IPFS gateway
   - Format timestamp as human-readable date

3. **Create PatientDashboard page** (`src/app/patient/page.tsx`)
   - Client-side page with wallet connection check
   - Show "Connect wallet" if not connected
   - Render UploadedDataList when connected
   - Include header with navigation

4. **Add AccessList component** (`src/components/AccessList.tsx`)
   - For each uploaded data item, show granted addresses
   - Query contract for access grantees (may need event indexing)
   - Show "No access granted" if empty

### Verification Criteria

- [ ] Page loads at `/patient` route
- [ ] Shows uploaded CIDs after wallet connects
- [ ] Timestamps are formatted as readable dates
- [ ] CID links open in IPFS gateway
- [ ] Access list shows granted addresses

### Must Haves

1. User sees all their uploaded data (CID + timestamp)
2. Each dataset shows which researchers have access
3. Loading states during data fetch
4. Error handling for contract call failures

### Technical Notes

- Contract: `getDataCount(userAddress)` returns total uploads
- Contract: `getData(dataId)` returns `{ cid, timestamp, metadata }`
- Access tracking may require listening to `AccessGranted`/`AccessRevoked` events
- Use wagmi's `useReadContract` for queries, `useContractEvent` for events
- IPFS gateway URL: `https://${gateway}/ipfs/${cid}`
